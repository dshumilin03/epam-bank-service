# epam-bank-service
Bank API - training project

# Functional Requirements:

- User Management system
- Money management and transaction payments
- Debit cards, credit cards, loans management
- Manager features(disabling users, look for user payments)
- Web interface

# Non-Functional Requirements:
- Maintainability (logging, tests, code quality etc)
- Usability (intuitive UI, internationalization)
- Reliability(Exception Handling)
- Security (all sensitive data must be secured)
# Use cases:
## User
- registers in bank system
- authorises in bank system
- changing his credentials
- opens debit and credit cards (with no percent first 90 days)
- getting loan
- Renew card
- paying debts
- Look for paying history
- Transfer money to other users
- Look for loan payment
